Unidad 4: Introducción a los Dispositivos Microcontrolados

Aplicaciones de la Electrónica Digital - 5º Año

1. ¿Qué es un Dispositivo Microcontrolado?

Un microcontrolador (MCU) es un circuito integrado que contiene en un solo chip todos los componentes de una pequeña computadora. Está diseñado específicamente para gobernar y controlar dispositivos electrónicos.

Microcontrolador vs. Microprocesador

Es crucial entender su diferencia:

  • Microprocesador (CPU): Es solo la unidad central de procesamiento. Necesita componentes externos (memoria RAM, memoria ROM, periféricos) para funcionar. Es el cerebro de una computadora personal.
  • Microcontrolador (MCU): Es un sistema completo. Integra la CPU, la memoria RAM, la memoria de programa (Flash ROM) y una variedad de periféricos en un único chip. Es el cerebro de un sistema embebido (como un lavarropas, un control remoto o un robot).

2. Arquitectura de un Microcontrolador

Aunque existen muchas familias de microcontroladores (PIC, AVR, ARM, etc.), todos comparten una arquitectura interna similar con los siguientes bloques fundamentales:

Componentes Internos

  • CPU (Unidad Central de Procesamiento): El núcleo que ejecuta las instrucciones del programa.
  • Memoria Flash (ROM): Memoria no volátil donde se almacena permanentemente el programa (el "firmware").
  • Memoria RAM (SRAM): Memoria volátil utilizada para almacenar las variables y datos temporales mientras el programa se ejecuta.
  • Periféricos: Son los módulos de hardware que permiten al microcontrolador interactuar con el mundo exterior. Los más comunes son:
    • GPIO (General Purpose Input/Output): Pines que se pueden configurar como entradas (para leer un pulsador) o salidas (para encender un LED).
    • ADC (Convertidor Analógico-Digital): Permite leer voltajes analógicos de sensores.
    • Timers/Contadores: Para medir tiempo, contar eventos o generar señales periódicas (PWM).
    • Módulos de Comunicación Serial: Como UART (para comunicación RS-232), I²C y SPI para comunicarse con otros chips.

3. Programación de Dispositivos Microcontrolados

Programar un microcontrolador implica escribir un conjunto de instrucciones que la CPU pueda ejecutar para realizar una tarea específica. El lenguaje más utilizado para esto es una extensión del Lenguaje C.

El Flujo de Trabajo de Desarrollo

  1. Escribir el Código: Se escribe el programa en un IDE específico para el microcontrolador (ej. MPLAB X para PICs, Arduino IDE para Arduino).
  2. Compilar: El compilador traduce el código C a código máquina específico para ese microcontrolador, generando un archivo binario (generalmente con extensión `.hex`).
  3. Grabar (o "Quemar"): Se utiliza un dispositivo de hardware llamado programador para transferir el archivo `.hex` desde la computadora a la memoria Flash del microcontrolador.
  4. Probar: Se alimenta el microcontrolador y este comienza a ejecutar el programa grabado.
