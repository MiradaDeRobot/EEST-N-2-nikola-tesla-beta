1. Lenguajes y Pseudolenguajes

Para comunicarnos con una computadora y darle instrucciones, necesitamos un lenguaje que ambos podamos entender. Los lenguajes de programación son sistemas formales diseñados para expresar procesos que pueden ser llevados a cabo por máquinas como las computadoras. Un algoritmo es una secuencia finita y ordenada de pasos para resolver un problema.

Niveles de Lenguajes

Los lenguajes se clasifican por su nivel de abstracción respecto al hardware de la computadora.

  • Lenguaje de Máquina: Es el nivel más bajo, compuesto por secuencias de 1s y 0s (código binario) que el procesador (CPU) entiende directamente. Cada tipo de procesador tiene su propio lenguaje de máquina. Es extremadamente difícil de leer y escribir para un humano.
  • Lenguaje Ensamblador (Assembly): Es un lenguaje de bajo nivel que utiliza mnemónicos (palabras cortas como 'MOV' para mover datos, 'ADD' para sumar, 'JMP' para saltar) para representar las instrucciones de máquina. Un programa llamado "ensamblador" lo traduce a código máquina. Sigue estando muy ligado a la arquitectura específica del procesador, pero es más legible que el binario.
  • Lenguaje de Alto Nivel: Son lenguajes como C, C++, Pascal, Delphi, Visual Basic, Python, etc. Utilizan una sintaxis más cercana al lenguaje humano, con palabras en inglés y notación matemática. Son independientes de la máquina (portables) y mucho más fáciles de aprender y usar. Un programa llamado "compilador" o "intérprete" se encarga de traducirlo a lenguaje de máquina.

Pseudocódigo y Diagramación Lógica

Antes de escribir código, es fundamental planificar la lógica del programa. Dos herramientas clave para esto son:

  • Pseudocódigo: Es una descripción de un algoritmo en lenguaje natural, pero con una estructura que se asemeja a un lenguaje de programación. No se puede compilar, solo sirve para planificar la secuencia de pasos de forma clara y sin preocuparse por la sintaxis exacta de un lenguaje.
  • Diagrama de Flujo: Es una representación gráfica del algoritmo. Utiliza símbolos estandarizados para representar acciones (rectángulos), decisiones (rombos), entrada/salida de datos (paralelogramos), inicio/fin (óvalos), etc., conectados por flechas que indican el flujo del programa. Es muy útil para visualizar la lógica de algoritmos complejos.

2. Introducción al Lenguaje C

El lenguaje C, desarrollado en los años 70, es uno de los lenguajes de programación más influyentes de la historia. Se le considera un lenguaje de "nivel medio", ya que combina la facilidad de un lenguaje de alto nivel (estructuras de control, variables) con la capacidad de acceder a recursos del sistema a bajo nivel (manejo directo de memoria con punteros), lo que lo hace ideal para la programación de sistemas operativos, drivers y, especialmente, para la electrónica y los sistemas embebidos.

Evolución Histórica e Importancia

C fue creado por Dennis Ritchie en los Laboratorios Bell. Su desarrollo estuvo íntimamente ligado al del sistema operativo UNIX, que fue reescrito casi en su totalidad en C. Esta simbiosis le dio una gran popularidad. En 1989, el Instituto Nacional Estadounidense de Estándares (ANSI) publicó un estándar para el lenguaje, conocido como ANSI C, para garantizar que un programa escrito en C pudiera compilarse y funcionar en diferentes plataformas sin cambios (portabilidad).

La influencia de C es inmensa. Lenguajes posteriores muy populares como C++ (que añade orientación a objetos), Java y C# heredaron gran parte de su sintaxis y estructura.

3. Entornos de Programación

Para escribir, compilar y probar un programa, se utiliza un Entorno de Desarrollo Integrado (IDE). Un IDE agrupa varias herramientas esenciales para el programador en una sola aplicación, facilitando el ciclo de desarrollo.

Compiladores y Linkeadores

El proceso para convertir el código fuente (lo que escribimos en un archivo .c) en un programa ejecutable (un archivo .exe en Windows) tiene varios pasos:

  • Preprocesador: Lee el código fuente y procesa las directivas que empiezan con '#' (como `#include` para añadir bibliotecas).
  • Compilador: Traduce el código fuente preprocesado a código objeto (.obj). El código objeto es una versión intermedia en lenguaje de máquina, pero aún no es un programa completo porque las llamadas a funciones de bibliotecas (como `printf`) no están resueltas.
  • Linkeador (Enlazador): Toma uno o más archivos de código objeto y los enlaza con el código de las bibliotecas de funciones necesarias. Resuelve las referencias a funciones y crea el archivo ejecutable final que el sistema operativo puede cargar y correr.
