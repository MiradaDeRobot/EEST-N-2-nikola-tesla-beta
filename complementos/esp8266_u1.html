Unidad 1: Fundamentos del ESP8266 y Conectividad Wi-Fi

Programación con ESP8266

1. ¿Qué es el ESP8266?

El ESP8266 es un microcontrolador de bajo costo con una capacidad fundamental integrada: conectividad Wi-Fi. Esto lo convierte en el cerebro ideal para proyectos de Internet de las Cosas (IoT), permitiendo que nuestros dispositivos se conecten a internet para enviar datos o ser controlados de forma remota.

Placas de Desarrollo Comunes: NodeMCU y Wemos D1 Mini

Aunque el chip es el ESP8266, rara vez se usa solo. Lo más común es utilizarlo en una placa de desarrollo que incluye el chip, un conversor USB-Serie para programarlo, un regulador de voltaje y los pines de E/S expuestos.

NodeMCU

Wemos D1 Mini

¡Atención al Voltaje! Aunque la placa se alimenta con 5V a través del conector USB, el chip ESP8266 y todos sus pines de entrada/salida (GPIO) funcionan a 3.3V. La placa incluye un regulador de voltaje que convierte los 5V del USB a los 3.3V que necesita el chip. Esto significa que conectar un sensor o componente de 5V directamente a sus pines de entrada puede dañarlo permanentemente. Siempre se deben usar adaptadores de nivel lógico.

2. Configuración del IDE de Arduino para ESP8266

Para poder programar el ESP8266, primero debemos indicarle al IDE de Arduino cómo hacerlo. Esto se hace añadiendo un "Gestor de Tarjetas" adicional, que contiene toda la información que el IDE necesita para compilar y subir el código a estas placas.

Pasos de Configuración

  1. Abre el IDE de Arduino y ve a `Archivo > Preferencias`.
  2. En el campo "Gestor de URLs Adicionales de Tarjetas", pega la siguiente URL: http://arduino.esp8266.com/stable/package_esp8266com_index.json
  3. Ve a `Herramientas > Placa > Gestor de Tarjetas...`.
  4. En el buscador, escribe "esp8266" y dale a instalar al paquete "esp8266 by ESP8266 Community".
  5. Una vez instalado, podrás seleccionar tu placa (ej. "NodeMCU 1.0") desde el menú `Herramientas > Placa`.

3. "Hola Mundo" del IoT: Escanear Redes Wi-Fi

Nuestro primer programa no hará parpadear un LED, sino que usará la capacidad principal del ESP8266: el Wi-Fi. Crearemos un sketch que escanee las redes Wi-Fi a su alrededor y las muestre en el Monitor Serie.

Análisis del Código

Este código utiliza la librería `ESP8266WiFi.h`, que ya viene instalada con el gestor de tarjetas.

#include "ESP8266WiFi.h" // Incluye la librería para el manejo de Wi-Fi

void setup() {
  Serial.begin(115200); // Inicia el puerto serie a una velocidad más alta
  WiFi.mode(WIFI_STA);
  WiFi.disconnect(); 
  delay(100);
  Serial.println("Setup finalizado. Iniciando escaneo...");
}

void loop() {
  Serial.println("Escaneando redes Wi-Fi...");
  int n = WiFi.scanNetworks();
  
  if (n == 0) {
    Serial.println("No se encontraron redes.");
  } else {
    Serial.print(n);
    Serial.println(" redes encontradas:");
    for (int i = 0; i < n; ++i) {
      Serial.print(i + 1);
      Serial.print(": ");
      Serial.print(WiFi.SSID(i));
      Serial.print(" (");
      Serial.print(WiFi.RSSI(i));
      Serial.print(")");
      Serial.println((WiFi.encryptionType(i) == ENC_TYPE_NONE) ? " " : "*");
      delay(10);
    }
  }
  Serial.println("");
  delay(5000);
}

4. Conectándose a una Red Wi-Fi

El siguiente paso lógico es conectarnos a nuestra propia red Wi-Fi. Esto es esencial para que nuestro dispositivo pueda acceder a internet.

Código para Conexión

Este sketch se conecta a una red específica e imprime la dirección IP que le asignó el router.

#include <ESP8266WiFi.h>

const char* ssid = "NOMBRE_DE_TU_RED";
const char* password = "PASSWORD_DE_TU_RED";

void setup() {
  Serial.begin(115200);
  delay(10);

  Serial.println();
  Serial.print("Conectando a ");
  Serial.println(ssid);

  WiFi.begin(ssid, password);

  while (WiFi.status() != WL_CONNECTED) {
    delay(500);
    Serial.print(".");
  }

  Serial.println("");
  Serial.println("¡Wi-Fi conectado!");
  Serial.print("Dirección IP: ");
  Serial.println(WiFi.localIP());
}

void loop() {
  // No se necesita código aquí para este ejemplo
}
