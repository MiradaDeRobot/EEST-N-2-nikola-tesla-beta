Unidad 3: Aplicaciones Avanzadas y FreeRTOS

Programación con ESP32

1. ¿Qué es un Sistema Operativo de Tiempo Real (RTOS)?

Un Sistema Operativo de Tiempo Real (RTOS) es un tipo de sistema operativo diseñado para gestionar el hardware y el software de un sistema embebido de forma predecible y determinista. Su función principal es administrar las diferentes tareas que debe realizar el microcontrolador, asignándoles prioridades y tiempo de procesador.

¿Por qué usar un RTOS en el ESP32?

El entorno de Arduino para ESP32 ya utiliza FreeRTOS por debajo. Aunque el `setup()` y el `loop()` nos lo ocultan, nos permite aprovechar el doble núcleo y gestionar tareas complejas de forma eficiente. Usar un RTOS es crucial cuando un proyecto necesita:

  • Realizar múltiples acciones simultáneamente (ej. leer un sensor, actualizar una pantalla y gestionar una conexión Wi-Fi al mismo tiempo).
  • Responder a eventos en un tiempo garantizado.
  • Evitar el uso de la función `delay()`, que bloquea por completo al procesador.

2. Actividad Práctica: Multitarea con FreeRTOS

Vamos a crear un programa que realice dos tareas en paralelo sin que una interfiera con la otra, demostrando el poder de la multitarea.

Análisis del Código

Este código crea dos tareas: una para hacer parpadear un LED cada 250 milisegundos y otra para leer un sensor de temperatura cada 2 segundos. Ambas se ejecutan en el mismo núcleo, pero el RTOS se encarga de alternar entre ellas.

#include "DHT.h"

#define LED_PIN 2
#define DHTPIN 4
#define DHTTYPE DHT11

DHT dht(DHTPIN, DHTTYPE);

// Tarea para el parpadeo del LED
void TareaBlink(void *pvParameters) {
  pinMode(LED_PIN, OUTPUT);
  for (;;) { // Bucle infinito de la tarea
    digitalWrite(LED_PIN, HIGH);
    vTaskDelay(250 / portTICK_PERIOD_MS); // Pausa de 250ms sin bloquear
    digitalWrite(LED_PIN, LOW);
    vTaskDelay(250 / portTICK_PERIOD_MS);
  }
}

// Tarea para leer el sensor
void TareaSensor(void *pvParameters) {
  dht.begin();
  for (;;) {
    float t = dht.readTemperature();
    if (!isnan(t)) {
      Serial.print("Temperatura: ");
      Serial.print(t);
      Serial.println(" *C");
    }
    vTaskDelay(2000 / portTICK_PERIOD_MS); // Pausa de 2 segundos sin bloquear
  }
}

void setup() {
  Serial.begin(115200);
  
  // Crear la tarea de parpadeo con prioridad 1
  xTaskCreate(
    TareaBlink,
    "Blink LED",
    1000,
    NULL,
    1,
    NULL);

  // Crear la tarea de lectura del sensor con prioridad 1
  xTaskCreate(
    TareaSensor,
    "Read Sensor",
    2000,
    NULL,
    1,
    NULL);
}

void loop() {
  // El loop principal queda vacío porque la lógica está en las tareas
}
Función Clave: `vTaskDelay()`
Esta función es la alternativa de FreeRTOS a `delay()`. La gran diferencia es que `vTaskDelay()` no detiene al procesador. Simplemente "duerme" la tarea actual y le permite al RTOS ejecutar otras tareas durante ese tiempo.
