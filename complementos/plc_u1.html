Unidad 1: Lógica de Escalera (Ladder) y Fundamentos de Control

Programación de PLC

1. ¿Qué es un PLC?

Un PLC (Controlador Lógico Programable) es una computadora industrial diseñada específicamente para automatizar procesos electromecánicos, como el control de maquinaria en una línea de montaje, atracciones de parques de diversiones o el control de semáforos. A diferencia de un Arduino o una Raspberry Pi, un PLC está construido para ser extremadamente robusto y fiable en entornos industriales hostiles, resistiendo vibraciones, ruido eléctrico y temperaturas extremas.

Componentes Clave de un PLC

  • CPU (Unidad Central de Procesamiento): El cerebro que ejecuta el programa de control. Realiza cálculos, toma decisiones lógicas y gestiona la comunicación entre los distintos módulos.
  • Módulos de Entrada (Inputs): Son los "sentidos" del PLC. Convierten las señales eléctricas de dispositivos de campo (pulsadores, sensores de proximidad, finales de carrera) en señales lógicas que la CPU puede entender.
  • Módulos de Salida (Outputs): Son las "manos" del PLC. Convierten las decisiones lógicas de la CPU en señales eléctricas para activar dispositivos externos como motores, luces, válvulas o contactores.
  • Fuente de Alimentación: Proporciona la energía estable y regulada (usualmente a 24V DC) que necesitan la CPU y los módulos para funcionar de manera fiable.

2. El Lenguaje de Escalera (Ladder)

El lenguaje de programación más común para los PLC es el Diagrama de Escalera o Ladder (LD). Su diseño se basa intencionadamente en los diagramas de circuitos de relés electromecánicos que los técnicos e ingenieros eléctricos han usado durante décadas. Esta familiaridad hizo que la transición de la lógica cableada a la lógica programada fuera mucho más sencilla.

Estructura y Flujo de Ejecución

  • Rieles de Alimentación: Dos líneas verticales a los lados que representan la fuente de energía virtual. La lógica fluye de izquierda a derecha.
  • Renglones (Rungs): Líneas horizontales donde se coloca la lógica. El PLC ejecuta los renglones en orden, de arriba hacia abajo, repetidamente.
  • Instrucciones: Símbolos que representan condiciones (entradas) y acciones (salidas).

El Ciclo de Scan (Scan Cycle)

El PLC no ejecuta el código de forma continua como un microcontrolador normal. Opera en un ciclo repetitivo llamado "scan":

  1. Leer Entradas: El PLC "congela" el estado de todas las entradas físicas y lo guarda en su memoria.
  2. Ejecutar Programa: Resuelve la lógica de cada renglón, de arriba a abajo, usando el estado de las entradas que guardó.
  3. Actualizar Salidas: Una vez resuelta toda la lógica, actualiza el estado de todas las salidas físicas al mismo tiempo.
  4. Repetir: Vuelve al paso 1. Este ciclo es extremadamente rápido, repitiéndose miles de veces por segundo.

3. Elementos Fundamentales de Ladder

La lógica de escalera se construye con tres elementos básicos que representan entradas, salidas y sus estados. Cada elemento está asociado a una dirección de memoria única que lo identifica (por ejemplo, `I:0/0` para la primera entrada, `O:0/1` para la segunda salida).

Contacto Normalmente Abierto (NO)

--| |--

Pregunta: "¿La entrada/memoria está en ON?"
Es "verdadero" (deja pasar la energía) solo si la entrada física o la memoria asociada está activada (estado 1). Ideal para un pulsador de "Marcha".

Contacto Normalmente Cerrado (NC)

--|/|--

Pregunta: "¿La entrada/memoria está en OFF?"
Es "verdadero" (deja pasar la energía) solo si la entrada física o la memoria asociada está desactivada (estado 0). Ideal para un pulsador de "Parada" o una parada de emergencia.

Bobina (Salida)

--( )--

Acción: "Activar la salida."
Se activa (se pone en estado 1) si la combinación lógica de contactos a su izquierda resulta verdadera.

4. Lógica de Control Básica y Enclavamiento

Combinando los elementos básicos, podemos crear toda la lógica de control necesaria para un automatismo, desde las compuertas lógicas más simples hasta complejos sistemas de control.

Compuertas Lógicas

Lógica AND (Contactos en Serie)

La Salida se activa solo si la Entrada_1 Y la Entrada_2 están activas. Útil para condiciones de seguridad donde dos cosas deben ocurrir a la vez.

|--| |------| |------( Salida )--|
| Entrada_1  Entrada_2          |

Lógica OR (Contactos en Paralelo)

La Salida se activa si la Entrada_1 O la Entrada_2 (o ambas) están activas. Útil para activar un proceso desde dos lugares distintos.

|--+--| |---+----( Salida )--|
|  Entrada_1  |                |
|     +--| |--+                |
|        Entrada_2               |

El Enclavamiento: La Memoria de 1 Bit

El enclavamiento (o auto-retención) es una de las técnicas más importantes en la programación de PLC. Permite que una salida permanezca activada incluso después de que el pulsador (que es momentáneo) que la activó se haya soltado. Es la forma más básica de crear una memoria.

En el ejemplo, al presionar "Start", la bobina "Motor" se activa. Inmediatamente, el contacto "Motor" (asociado a esa bobina) se cierra, creando un camino alternativo para la energía. Ahora, aunque se suelte "Start", la energía sigue fluyendo a través del contacto "Motor", manteniéndolo encendido. La única forma de romper este sello es activando el contacto NC "Stop".

|--| |------+--|/|------( Motor )--|
|  Start    |    Stop             |
|           |                     |
|--| |------+                     |
|  Motor                          |
