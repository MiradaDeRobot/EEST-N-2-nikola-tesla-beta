Unidad 3: Internet de las Cosas (IoT) con MQTT

Programación con ESP8266

1. ¿Qué es el Internet de las Cosas (IoT)?

El Internet de las Cosas (IoT) se refiere a la red de objetos físicos ("cosas") que llevan incorporados sensores, software y otras tecnologías con el fin de conectarse e intercambiar datos con otros dispositivos y sistemas a través de internet. El ESP8266 es un componente fundamental en el mundo del IoT.

Ejemplos de Aplicaciones IoT

  • Hogar Inteligente (Smart Home): Luces que se encienden solas, termostatos que aprenden tus hábitos, cerraduras que puedes abrir con tu celular.
  • Ciudades Inteligentes (Smart Cities): Sensores de estacionamiento que indican lugares libres, sistemas de recolección de basura que optimizan sus rutas, alumbrado público que se regula según el tráfico.
  • Industria 4.0: Sensores en máquinas que predicen fallas antes de que ocurran, sistemas de logística que rastrean paquetes en tiempo real.

2. El Protocolo MQTT

Mientras que un servidor web es útil para el control local, no es eficiente para que muchos dispositivos envíen datos a un servidor central. Para esto, se utiliza un protocolo diseñado para IoT: MQTT (Message Queuing Telemetry Transport).

Modelo Publicador/Suscriptor

MQTT es un protocolo de mensajería extremadamente ligero que funciona con un modelo de publicación y suscripción. Los componentes son:

  • Broker: Es un servidor central que recibe todos los mensajes y los distribuye a los clientes interesados.
  • Cliente Publicador (Publisher): Un dispositivo (como un ESP8266 con un sensor) que envía mensajes a un "tema" (topic) específico en el broker.
  • Cliente Suscriptor (Subscriber): Un dispositivo o aplicación que se suscribe a un "tema" en el broker. Cuando un publicador envía un mensaje a ese tema, el broker se lo reenvía a todos los suscriptores.
Temas (Topics): Son como "canales" de información, organizados jerárquicamente. Por ejemplo, un sensor de temperatura en la cocina podría publicar en el tema `casa/cocina/temperatura`. Un cliente podría suscribirse a `casa/cocina/#` para recibir todos los datos de la cocina.
Calidad de Servicio (QoS): MQTT permite definir la fiabilidad de la entrega de un mensaje con tres niveles de QoS (0, 1 y 2), desde "enviar y olvidar" hasta "asegurar que llegue una sola vez".

3. Actividades Prácticas con MQTT

Actividad 1: Enviando Datos de un Sensor (Publicador)

Vamos a construir un termómetro IoT que publique la temperatura y humedad leídas por un sensor DHT11 a un broker MQTT público en internet.

Este código utiliza la librería `PubSubClient`, que se debe instalar desde el Gestor de Librerías del IDE de Arduino.

#include <ESP8266WiFi.h>
#include <PubSubClient.h>
#include "DHT.h"

// --- Configuración Wi-Fi y MQTT ---
const char* ssid = "NOMBRE_DE_TU_RED";
const char* password = "PASSWORD_DE_TU_RED";
const char* mqtt_server = "broker.hivemq.com"; // Broker MQTT público

// --- Inicialización de objetos ---
WiFiClient espClient;
PubSubClient client(espClient);
DHT dht(D4, DHT11); // Sensor DHT11 en el pin D4

void setup() {
  Serial.begin(115200);
  dht.begin();
  
  // Conexión a Wi-Fi
  WiFi.begin(ssid, password);
  while (WiFi.status() != WL_CONNECTED) {
    delay(500);
    Serial.print(".");
  }
  Serial.println("\nWi-Fi conectado!");

  // Conexión al Broker MQTT
  client.setServer(mqtt_server, 1883);
}

void loop() {
  if (!client.connected()) {
    reconnect();
  }
  client.loop();

  float t = dht.readTemperature();
  if (!isnan(t)) {
    client.publish("escuela/tesla/temperatura", String(t).c_str());
    Serial.print("Temperatura enviada: ");
    Serial.println(t);
  }

  delay(10000); // Espera 10 segundos
}

void reconnect() {
  while (!client.connected()) {
    Serial.print("Intentando conexión MQTT...");
    if (client.connect("ESP8266_SensorTemp")) {
      Serial.println("conectado");
    } else {
      Serial.print("falló, rc=");
      Serial.print(client.state());
      Serial.println(" intentando de nuevo en 5 segundos");
      delay(5000);
    }
  }
}

Actividad 2: Control Remoto de un LED (Suscriptor)

Ahora haremos lo contrario: nuestro ESP8266 se suscribirá a un tema y esperará comandos ("on" o "off") para encender o apagar un LED.

Para probarlo, puedes usar un cliente MQTT online como HiveMQ Websocket Client para publicar "on" o "off" en el tema `escuela/tesla/luz`.

#include <ESP8266WiFi.h>
#include <PubSubClient.h>

// ... (Configuración Wi-Fi y MQTT igual que en el ejemplo anterior) ...
const int pinLed = D1; // LED conectado al pin D1

void callback(char* topic, byte* payload, unsigned int length) {
  Serial.print("Mensaje recibido en [");
  Serial.print(topic);
  Serial.print("] ");
  
  String mensaje;
  for (int i = 0; i < length; i++) {
    mensaje += (char)payload[i];
  }
  Serial.println(mensaje);

  if (mensaje == "on") {
    digitalWrite(pinLed, HIGH);
  } else if (mensaje == "off") {
    digitalWrite(pinLed, LOW);
  }
}

void setup() {
  pinMode(pinLed, OUTPUT);
  // ... (Setup de Serial y Wi-Fi igual que en el ejemplo anterior) ...
  
  client.setServer(mqtt_server, 1883);
  client.setCallback(callback); // Asigna la función que se ejecutará al recibir un mensaje
}

void reconnect() {
  while (!client.connected()) {
    Serial.print("Intentando conexión MQTT...");
    if (client.connect("ESP8266_ControlLED")) {
      Serial.println("conectado");
      // Se suscribe al tema para recibir comandos
      client.subscribe("escuela/tesla/luz");
    } else {
      // ... (código de reintento igual que antes) ...
    }
  }
}

void loop() {
  if (!client.connected()) {
    reconnect();
  }
  client.loop(); // Mantiene la conexión y revisa si hay mensajes nuevos
}
